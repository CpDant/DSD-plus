Data Smell Documentation

Below you can see the different data smells divided by four categories. In every tab the available data smells for the data smell detection are listed. Furthermore, you can see the parameters for every single data smell. In the customation phase, you are able to choose a tolerant, medium or strict mode for the presettings. In addition, the data smells are further divided into the categories minority and majority faulty. Down below you can see the the default values for the presettings. For minority faulty data smells, a small percentage of faulty column values suffices for a column to contain a data smell. For majority faulty data smells, the majority of the column values must be faulty for a column to contain a data smell.

Available Smells

Duplicated Value Smell Maj

This smell occurs when data values are syntactically equal across several data instances.
Example: 12, 2, 2, 2, 2, 2, 2, 2, 2, 3, 6, 13


Extreme Value Smell Maj

This smell occurs when data instances have extreme data values relating to other data instances.
Example: 1, 5, 10, 12, 6, 7, 2300, 13, 17
Parameters
threshold: The threshold to use regarding the z-score. This parameter can be configured by users but is set to 3 by default. It is assumed that this parameter is a positive number.

Suspect Sign Smell Maj

This smell occurs when data instances have a different sign (+/-) than the rest of the instances.
Example: 2, 7, 22, 13, 95, -5, 65, 20
Parameters
percentile_threshold: This parameter must be in the interval [0,1]. It controls which quantiles are computed. The computed quantiles are used to determine whether the majority of the column values are positive or negative.

More Smells

Dummy Value Smell

This smell characterizes a situation in which a kind of substitute value (sentinel value) is used due to several reasons (e.g. missing values, unknown values, computation errors, surpass "not NULL" constraints etc.).
Example: 1970-01-01 (Unix epoch), 999

Meaningless Value Smell

This smell arises when a data value has no common meaning or contains suspect repeating sequences of characters.
Example: £$1%S13, aaa, abcabc, 123412341234

Misspelling Smell

This smell arises when a data value probably has a spelling error.
Example: combuter, sofdware, 7O34 ‘O’ instead of ‘0’

Suspect Class Value Smell

This smell characterizes a situation in which categorical data instances have uncommon class values.
Example: high, high. high, low, low, sunny, high

Suspect Date Value Smell

This smell occurs when a data value represents a date far in the past or future.
Example: 1899-03-22, 2099-03-02

Suspect Date/Time Interval Smell

This smell occurs when there is a very long/short date/time interval between data instances.
Example: 2019-01-01, 2019-01-02, 2019-10-01, 2019-10-02

Suspect Distribution Smell

This smell characterizes a situation in which the data values have a suspect distribution.
Example: 1, 3, 4, 5, 3, 4, 3, 500, 15000, 780000, 800000, 800000

Available Smells

Integer as String Min

This smell occurs when an integer is encoded as a string.
Example: "3"

Floating-point Number as String Min

This smell occurs when a floating-point number is encoded as a string.
Example: "3.5"

Integer as Floating-point Number Min

This smell occurs when an integer is encoded as a floating-point number.
Example: 3.0
Parameters
epsilon: The threshold for the absolute difference of a floating point number. to the nearest integer. This parameter must be a floating point value greater than zero.

More Smells

Date as DateTime

This smell occurs when a date is encoded as a datetime data type.
Example: 2020-12-30 00:00:00

Date as String

This smell occurs when a date is encoded as a string.
Example: "2020-12-30"

DateTime as String

This smell occurs when a date and a timestamp are encoded as a string.
Example: "2020-12-30 11:50:40"

Time as String

This smell occurs when a timestamp is encoded as a string.
Example: "11:50:40"

Available Smells

Casing Maj

This smell occurs when data values represent an unusual use of upper and lower case (Mixed Case, Upper Only, Lower Only).
Example: GoOD MorNiNg, GOOD MORNING, good morning

Long Data Value Maj

This smell arises when data values are too long to understand.
Example: Pneumonoultramicroscopicsilicovolcanoconiosis, 12Am12-12-20highpressalarmset0
Parameters
length_threshold: The minimum number of characters a word must consist of to be considered "long". This parameter must be an integer greater than zero.

Missing Value Maj

This smell arises when data instances have no data values.
Example: 12, 2, 3, , , 4, 55

More Smells

Ambiguous Date/Time Format

This smell occurs when a date is represented in short format or a timestamp is represented in 12 hour format.
Example: 06-03-05 vs. date: 06-05-03, 05:13

Ambiguous Value

This smell arises when data values represent abbreviations, homonyms, acronyms or an ambiguous context.
Example: Dr (doctor vs. drive), express (done fast vs. show thoughts with words), address (speak to vs. location), Miami (Florida vs. Ohio)

Contracting

This smell occurs when data values represent shortened version of words or phrases.
Example: don't, I'd

Extraneous Value

This smell occurs when data values provide additional, likely unnecessary information.
Example: CJ. John Roberts Jr., Fri Mar 05 2021 17:27:40 GMT+0100

Intermingled Data Type

This smell characterizes a situation in which data values contain numeric and alphabetic characters.
Example: Room 12, 90 Days

Separating

This smell arises when data values contain thousands separators (space, underbar, dot etc.) for grouping digits. The usage of delimiters can result in ambiguity when also a decimal separator is used and lead to problems decoding the values.
Example: 23 432, 21.321, 12 340 000

Spacing

This smell arises when data values contain an uncommon pattern of spaces (Trailing Space, Leading Spaces, Multiple Spaces, Missing Spaces).
Example: “John Smith “, " John Smith”, ” John Smith “, "JohnSmith"

Special Character

This smell occurs when data values contain special characters (non-alphanumeric) like Commas, Dots, Hyphens, Apostrophes, Tab Char, Punctuation, Parentheses, Dashes, accented Letters, etc.
Example: "Hello, how are you - today", "New.York.", "John's house"

Synonym

This smell occurs when data values have the same semantic meaning but are syntactically different (e.g. Aliases, Nick names, Pseudonyms).
Example: (intelligent, clever, smart), (Bill Clinton, President Clinton, William Jefferson Clinton)

Tagging

This smell occurs when data values represent tags.
Example: \begin{itemize}, <head>

More smells

Abbreviation Inconsistency

This smell occurs when abbreviations or contractions are not used consistently.
Example: Dr. Hofer vs. Doctor Huber, I'd vs. I would

Casing Inconsistency

This smell characterizes a situation in which upper and lower case is not used consistently.
Example: How are you?, fine., and you? Great.

Class Inconsistency

This smell arises when class values for categorical data instances are not used consistently (e.g. different representations or abstractions).
Example: male, male, women, 1, 1, male, frozen pizza, frozen food

Date/Time Format Inconsistency

This smell occurs when date or time formats are not used consistently.
Example: 2020-01-30, 30-01-2020, 20-01-30, 05:13, 05:13:25

Missing Value Inconsistency

This smell characterizes a situation in which constants are not used consistently to represent missing data values.
Example: 12, NaN, Na, NULL, 22, inf, , 23

Separating Inconsistency

This smell occurs when thousands separator are not used consistently.
Example: 23 100, 23.500, 25_000, 27 000

Spacing Inconsistency

This smell occurs when spacing is not used consistenly.
Example: +43 664 123, +43664 546

Special Character Inconsistency

This smell occurs when special characters are not used consistently.
Example: (2), (44), [32], (32)

Syntax Inconsistency

This smell characterizes a situation in which the general syntax (decimal places, telephone area codes, extraneous information etc.) of data values is not used consistently. Mainly, all other consistency issues which do not fall in one of the other categories related to this category.
Example: 32.2, 44.1111, +43650123, 0043654124, Miss Smith, Miss Taylor, Mayer, Miss Stone

Unit Inconsistency

This smell arises when units of measurement are not (implicitly) used consistently.
Example: 1.80, 1.84, 190, 176, 1.6

Transposition Inconsistency

This smell arises when ordering of words or special characters is not used consistently.
Example: €10.5 vs. 11.0€, Jack Smith vs. Stevens John
